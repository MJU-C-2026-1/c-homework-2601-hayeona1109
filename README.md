# 💻 [2026-01] 나만의 C언어 소프트웨어 개발 프로젝트

## 1. 시나리오 제목
알바생을 위한 스마트 급여 계산 및 근무 스케줄 관리 시스템

<br>


## 2. 시나리오 (5~10줄)
이 프로그램은 아르바이트생이 매일의 근무 데이터와 시급 정보를 입력하여 예상 급여를 체계적으로 관리할 수 있도록 돕는 개인 맞춤형 급여 관리 시스템입니다.  
사용자는 자신이 근무하는 알바 장소의 이니셜(문자)과 기본 시급(정수), 실제 근무 시간(정수), 그리고 지원받는 식대(실수)를 입력합니다.  
시스템은 입력된 데이터를 바탕으로 당일의 총합 예상 일급을 사칙연산을 통해 정확히 산출하여 출력합니다.  
향후 업데이트를 통해 주휴수당 지급 대상 여부를 판별하는 조건문 로직과 반복문을 활용한 메뉴 선택 시스템을 추가할 예정입니다.  
최종적으로는 여러 개의 알바 장소를 배열로 저장하고 관리할 수 있는 통합 스케줄링 솔루션으로 완성하는 것을 목표로 합니다.   

<br>


## 3. 예상 기능 및 메뉴 (최소 5개)
1.	근무 정보 입력 (브랜드 이니셜, 시급, 시간, 식대)  
2.	당일 총 급여 계산  
3.	주휴수당 및 추가 수당 판별 (V2.0 예정)  
4.	월별 급여 합산 조회 (V3.0 예정)  
5.	프로그램 종료

## 🟦 [1차 과제: V1.0] 기초 뼈대 구축 (입출력과 변수)
### **✨V1.0 개발 내용:**
• 사용자로부터 알바 브랜드 이니셜(문자), 시급/근무 시간(정수), 식대 지원비(실수)를 입력받아 당일 예상 일급을 계산하여 출력합니다.  
• 알바 장소의 특성을 살려 문자, 정수, 실수 등 3가지 이상의 자료형을 사용하여 입출력 화면을 구현했습니다.  
• 사칙연산을 활용해 '(시급 * 시간) + 식대'를 계산하여 최종 일급을 산출했습니다.

## 🟩 [2차 과제: V2.0] 조건 분기 적용 (조건문) - 향후 작성 예정
### **✨2차 과제 업데이트 내용:**
  * 내용.
  * 이번 V2.0 업데이트의 핵심은 '단순 계산'에서 '지능적 판단'으로의 진화입니다. 기존에는 사용자가 입력한 값을 그대로 곱하여 결과를 냈다면, 이번 버전에서는 if-else 조건문을 도입하여 다음 세 가지 핵심 기능을 추가했습니다.

근로기준법 준수: 1일 8시간 초과 근무 시 1.5배의 연장 수당이 자동으로 계산되도록 로직을 설계했습니다.

주휴수당 가이드: 주간 총 근무 예정 시간을 입력받아, 15시간 이상일 경우 주휴수당 지급 대상임을 안내하는 메시지 출력 기능을 추가했습니다.

데이터 무결성 검증: 시급이나 근무 시간에 음수값이 입력될 경우, 계산을 진행하지 않고 경고 메시지를 띄우는 예외 처리 로직을 구현했습니다.
    
### **🤖 AI 파트너십 과정**
🤖 AI 파트너십 과정
내용 1

프롬프트 요약: "기존 급여 계산기 코드에 '8시간 초과 근무 시 연장 수당 1.5배 적용'과 '주 15시간 이상 근무 시 주휴수당 대상 알림' 기능을 if-else문으로 구현하는 방법 제안 요청"

적용 내용: if (working_hours > 8) 조건을 통해 연장 근로 수당(extra_pay)을 분리 계산하고, 주간 총 근무 시간을 입력받아 주휴수당 지급 대상 여부를 텍스트로 출력하는 판단 로직을 구축함.

내용 2

프롬프트 요약: "사용자가 잘못된 숫자(음수)를 입력하거나 메뉴 선택 시 범위를 벗어난 값을 입력할 경우를 대비한 방어적 프로그래밍 코드 검토 요청"

적용 내용: 논리 연산자(||)를 활용해 if (hours < 0 || wage < 0)와 같은 예외 처리 구문을 추가하여 프로그램의 안정성을 높임.
    
### **🛠️ Troubleshooting & 기술 회고:**
  문제 1: 입력 버퍼로 인한 문자 입력 건너뛰기 발생

상황: scanf로 시급(정수)을 입력받은 후, 다시 브랜드 이니셜(문자)을 입력받으려 할 때 프로그램이 입력을 받지 않고 넘어가는 현상 발생.

해결: scanf(" %c", &brand_initial);와 같이 %c 앞에 공백을 추가하여 버퍼에 남은 엔터(\n) 값을 무시하도록 처리하여 해결함.

문제 2: 조건문 내 연산 우선순위 오류

상황: hourly_wage * working_hours * 1.5와 같은 식에서 형변환이 제대로 이루어지지 않아 소수점 금액이 잘리는 문제 발생.

해결: 계산 결과가 저장될 total_day_pay를 float으로 선언하고, 계산식에 (float) 명시적 형변환을 사용하여 원 단위 미만 오차를 방지함.
