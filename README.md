## 2. 시나리오 (5~10줄)
이 프로그램은 아르바이트생이 매일의 근무 데이터와 시급 정보를 입력하여 예상 급여를 체계적으로 관리할 수 있도록 돕는 개인 맞춤형 급여 관리 시스템입니다.  
사용자는 자신이 근무하는 알바 장소의 이니셜(문자)과 기본 시급(정수), 실제 근무 시간(정수), 그리고 지원받는 식대(실수)를 입력합니다.  
시스템은 입력된 데이터를 바탕으로 당일의 총합 예상 일급을 사칙연산을 통해 정확히 산출하여 출력합니다.  
향후 업데이트를 통해 주휴수당 지급 대상 여부를 판별하는 조건문 로직과 반복문을 활용한 메뉴 선택 시스템을 추가할 예정입니다.  
최종적으로는 여러 개의 알바 장소를 배열로 저장하고 관리할 수 있는 통합 스케줄링 솔루션으로 완성하는 것을 목표로 합니다.

* **사용자 메뉴 및 구조적 프로그래밍 적용:** 일회성 실행에 그치던 기존 스크립트를 `while(1)` 무한 루프 구조로 묶고 `switch-case` 제어문 인터페이스를 설계하여, 사용자가 '4. 프로그램 종료'를 선택해 `break`를 실행하기 전까지 계속 반복 사용할 수 있도록 편의성을 극대화했습니다.
* **독립적 모듈화를 위한 함수 정의 및 호출:** `main` 함수에 집중되어 있던 코드를 역할에 따라 최소 3개 이상의 개별 함수(`input_data`, `calculate_pay`, `print_and_accumulate`)로 완전 쪼개어 분리했습니다. 이로 인해 메인 루프 내부 코드는 마치 한 눈에 읽히는 '목차'처럼 간결하게 정돈되었습니다.
* **매개변수 및 반환값의 유기적 매핑:** 포인터 변수 주소 및 값 기반의 매개변수(인자)를 전달받아 처리하는 인터페이스를 확립하였으며, 연산 함수에서는 처리 결과인 일일 급여 데이터를 `return` 구문을 통해 명확한 실수형(`float`)으로 반환하도록 세부 타입을 일치시켰습니다.
* **전역 변수 관리를 통한 데이터 누적:** 프로그램 전역 스코프에 `g_total_monthly_pay`(누적 급여)와 `g_total_monthly_hours`(누적 시간) 변수를 선언했습니다. 이를 통해 루프가 반복 실행되어도 정보가 휘발되지 않고, 출력 함수 내부에서 이 전역 변수를 직접 참조하여 매번 데이터를 가산 및 누적 업데이트할 수 있는 지속성 로직을 갖추었습니다.

### **🛠️ Troubleshooting & 기술 회고:**
1. **문제 1: 입력 제어 분기에서 숫자가 아닌 문자를 오입력했을 때 입력 버퍼 정체로 인한 메뉴판 무한 렌더링 현상** * **상황:** 사용자가 메뉴 선택지에 정수형 데이터 대신 'a'와 같은 잘못된 타입의 문자를 타이핑했을 때, `scanf` 버퍼에 해당 기호가 잔류하면서 조건 비교 로직을 무한 스킵하고 화면을 엉망으로 도배하는 런타임 에러가 발생함.  
   * **해결:** `if (scanf("%d", &menu_choice) != 1)` 구문 조건식을 작성하여 입력 성공 여부를 검증하고, 실패 감지 시 스트림 내 찌꺼기를 날려주는 `while(getchar() != '\n');` 버퍼 소거 코드를 긴급 배치하여 입력 안정성을 획득함.  
2. **문제 2: 함수화 과정에서 매개변수 데이터 바인딩 및 타입 미스매치로 인한 값 잘림 현상** * **상황:** 연비 식대나 가산 비율(`0.5`)을 연산하는 함수 내부에서 실수형 리터럴 연산을 진행할 때, 명시적 형변환을 빠뜨리거나 매개변수 선언부가 꼬이면서 소수점 아래 금액 데이터들이 유실 및 왜곡되어 원 단위 미만에서 오차가 생김.  
   * **해결:** 돈을 다루는 자료형 연산의 엄밀성을 지키기 위해 가산 비율 상수에 실수 플래그(`0.5f`)를 마킹하고 계산 함수 프로토타입 설계 시 명시적 캐스팅(`(float)`) 처리를 더해 완벽한 연산값을 호출처로 반환할 수 있도록 디버깅 완료함.

🤖 AI 파트너십 과정
내용 1

프롬프트 요약: "기존 단일 입력 시스템을 여러 알바 장소를 등록하고 선택해 입력할 수 있도록 구조체와 배열을 활용한 데이터 확장 설계 구조 제안 요청"

적용 내용: struct Store를 정의하여 복수의 데이터 필드를 유기적으로 묶고, 메인 루프에서 배열 인덱스를 통해 각 사업장 데이터에 접근하는 인프라를 구축함.

내용 2

프롬프트 요약: "구조체 배열에 누적된 여러 사업장의 데이터를 포인터와 반복문으로 안전하게 순회하며, 전체 월별 통계를 가시성 좋게 출력하는 리포트 함수 리팩토링"

적용 내용: 배열의 유효 등록 개수를 카운트하는 g_store_count 변수를 활용해 빈 배열 요소 접근을 차단하고, formatted output을 활용한 깔끔한 표 형태의 통계 창을 구현함.

🛠️ Troubleshooting & 기술 회고:
문제 1: 구조체 배열 인덱스 초과 및 잘못된 메모리 접근 제어

상황: 사용자가 등록된 알바 장소의 범위를 벗어나거나 제한 개수(5개)를 초과하여 데이터를 추가하려 할 때 프로그램이 강제 종료되거나 덤프 데이터가 출력되는 현상 발생.

해결: 데이터를 입력받기 전 if (g_store_count >= 5) 예외 처리 검증 구문을 배치하여 배열 오버플로우를 원천 차단하고 안정성을 확보함.

문제 2: 구조체 포인터 매개변수 전달 시 연산자 우선순위 오류

상황: 구조체 배열의 주소를 함수로 넘겨 값을 변경할 때, *pointer.variable 형태로 작성하여 멤버 접근 연산자(.)가 참조 연산자(*)보다 우선순위가 높아 컴파일 에러 발생.

해결: 구조체 포인터 접근 연산자인 화살표 연산자(->)를 사용하여 pointer->variable 구조로 코드를 정정함으로써 가독성과 연산 정확성을 모두 해결함.
